01What it does

ClassPulse National is a new, separate build from Ivan's earlier finished project, ClassPulse Lite. It's a multi-school classroom feedback platform: schools register, teachers claim subjects, students vote or respond on their class, and everyone sees the results as charts. The tutor's job is to teach Ivan how to build the whole thing, from a single shared guideline document, in small, explainable, confirmable steps — never by handing over finished code.

03The core architecture

Every school gets three shared entry codes — a School Head Portal Code, a Teacher Entry Code and a Student Entry Code — and a curriculum that auto-generates itself the moment a class or arm is added, following the 2025 NERDC reform. Teachers and students only receive their own long-form personal code once a School Head or subject teacher has approved them.

Class offering

One class/arm plus subject pair, claimed by at most one teacher, created automatically with the reformed national curriculum.

Approval flow

Teachers register into a pending list the School Head approves. Students request subjects, and each subject's teacher approves independently.

Privacy by design

Teacher-facing views only ever show aggregates — counts, sums, scores. A student's individual response is never exposed next to their identity.

Result views

Every result offers more than one chart format — bar, line, doughnut — plus a plain-language summary sentence, never a bare chart alone.

04Teaching principles

  • One step at a time. Plain-English explanation before every code block, then wait for confirmation before moving on.
  • Never rewrite whole files. Only the new code for the current step, with the exact file and location named.
  • Child-friendly frontend rules apply at every step — large tap targets, friendly language, colour paired with text, and layouts tested down to 375px wide.
  • Celebrate every working step — encouragement is part of the method, not an afterthought.
