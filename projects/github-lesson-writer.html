GitHub Lesson Writer

Hand it a lesson number and a set of source links, and it fetches, merges and rebuilds them into one complete, zero-prior-knowledge Markdown lesson — never a stitched-together summary of the sources.

Input
Lesson number + source URLs
Output format
Markdown, fixed front matter
Audience
Zero prior-knowledge learners
Naming
lesson-[N]-[topic].md

01What it does

Every source link is fetched in the order it was given, in full, before a single word of the lesson is written. Where two sources cover the same idea, they're merged into one clearer explanation rather than repeated. The result reads as one smooth, logically progressive lesson — not a set of disconnected excerpts — and is grounded throughout in Nigerian names, cities and everyday scenarios.

02The three-phase teaching pattern

  • Build understanding first. Every technical term is defined in plain language, paired with an analogy, and shown with the simplest possible example — followed immediately by a common beginner mistake and its correction.
  • Guided practice. Each exercise runs Objective → Scenario → Steps → Hints → Expected Output → Self-Check, set in realistic scenarios like Lagos traffic or school results, with an optional "what-if" variation at the end.
  • Mini project. One project combining the lesson's concepts, structured Setup → Core Logic → Enhancements → Final Output, with a milestone shown after every stage.

03Fixed lesson structure

Every file opens with a required YAML front matter block, then follows the same thirteen-part order every time: introduction and objectives, prerequisite recap, numbered concept sections, standalone code examples with expected output, three guided exercises, code challenges, a milestone-based mini project, common mistakes, reflection questions, a completion checklist and a quick-reference card.

Code examples

Every concept opens with one isolated, 3–6 line example, growing from very simple to realistic, each followed by its expected browser output.

Quality checklist

Fourteen checks — from front matter present to gradually increasing difficulty — verified before any file is saved.
